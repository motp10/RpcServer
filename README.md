# SDN FDB Синхронизатор

Центральный координатор для управления таблицами MAC-адресов в VXLAN-сетях.

### Функционал:

* Синхронизация: Сбор FDB-обновлений от VTEP и их мгновенная рассылка всем участникам сегмента (VNI).

* Анализ мобильности: Обнаружение «флапающих» MAC-адресов и предотвращение сетевых петель.

* Подавление дубликатов сообщений и ограничение скорости (Rate Limiting) для защиты VTEP.

* Надежность: Хранение состояния на диске для быстрого восстановления после сбоев.

### Стек: 
C++, gRPC, Protobuf.


## Входные данные: Получение FDB обновлений от коммутаторов, содержащих:

        Hostname (идентификатор коммутатора)

        IP-адрес VTEP

        VNI (VXLAN Network Identifier, от 1 до 16 миллионов)

        MAC-адрес(а)

        Тип операции: ADD (добавление) или DELETE (удаление)

## Конвейер обработки:

Сохранение/обновление в локальной базе данных

Определение всех VTEP в том же VNI

Распространение обновления на все релевантные VTEP (кроме источника)

# Ключевые возможности
* Сохранность данных

* Дисковое хранилище для восстановления после сбоев

* Быстрый перезапуск с восстановлением состояния

* Поддержка полных дампов таблицы и запросов по конкретному VTEP/VNI

## Обнаружение флапающих MAC-адресов

Мониторинг мобильности MAC-адресов в рамках одного VNI

Обнаружение частой смены VTEP (потенциальные сетевые петли)

Автоматическая маркировка нестабильных MAC и отправка инструкций на блокировку

Поддержка состояний MAC: ACTIVE (стабильный) и UNSTABLE (флапающий)

## Защита производительности

Rate limiting: Предотвращение перегрузки VTEP

Подавление дубликатов: Игнорирование повторяющихся идентичных событий от одного источника

Упорядоченная обработка: Сохранение последовательности сообщений
